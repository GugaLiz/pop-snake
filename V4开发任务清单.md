## V4 新玩法分支：变色蛇头·颜色匹配（发光提示 + 限吃同色）

### 玩家视角一句话
蛇头会周期性变色；当蛇头变成某个颜色时，棋盘上同色方块会发光提示；此时蛇只能吃同色方块。

### 设计意图
- 把“规划/选择”引入到经典吃块循环：玩家要根据颜色节奏与地图分布做路线决策
- 用强反馈（蛇头变色 + 同色发光）降低理解成本，让玩家第一眼就知道“现在该吃什么”

---

## 核心规则（可配置，默认按本玩法分支取值）

### 1) 颜色系统
- `colorSet`：本关/本模式启用的颜色集合（推荐从 2 色起步：红/黄；后续 3~5 色逐步增加）
- `headColor`：蛇头当前颜色（属于 `colorSet`）

### 2) 蛇头变色
- 自动变色：每隔 `headColorSwitchIntervalMs` 切换一次（参考 3000ms → 2500ms → 2000ms → 1500ms → 1000ms）
- 切换方式：
  - 初期可用固定循环（红→黄→蓝→…）
  - 后期可切换为无规律随机（避免玩家纯背节奏）
- 反馈要求：
  - 蛇头颜色变化必须有“瞬间可感知”的动画（例如短促描边闪烁/扩散光圈/缩放回弹）
  - HUD 需要同步显示“当前颜色”（色点/色条/倒计时环）

### 3) 方块与发光提示
- 棋盘会同时存在多种颜色方块
- 当 `headColor = X`：
  - 所有颜色为 `X` 的方块进入“可吃状态”，并出现轻微呼吸外发光（glow/pulse）
  - 非 `X` 方块保持普通显示（不发光或明显弱于可吃状态）

### 4) 吃块规则（V4 默认：严格同色可吃）
- 吃到同色方块（方块颜色 == `headColor`）：
  - 蛇身增长 1 节（新增节颜色 = 方块颜色）
  - 得分增加（建议 +10）
  - 正反馈（短浮字/音效/小粒子）
- 触碰异色方块（方块颜色 != `headColor`）：
  - **默认方案（严格）：判定为失败**（理由：最直观地兑现“只能吃同色”）
  - 兼容方案（可选配置，用于后续调参/关卡多样性）：
    - `penalty_shorten`：蛇身缩短 1 节（最低保留 3 节）且扣分（建议 -5），并强制立刻变色 1 次
    - `penalty_stun`：短暂僵直（例如 500~1000ms）+ 立刻变色 1 次
    - `penalty_block`：异色方块不可被吃掉（保留在原地），蛇头不能进入该格（等价“撞墙失败”或“强制停顿”——需二选一）

> 说明：上面“默认失败”与“惩罚版”请都保留为可配置项；V4 分支默认用严格版，以符合本次补充描述。

## 关卡/难度参数（第一类玩法 1-5 关的落地字段）

即使当前工程还没有完整“关卡系统”，也建议先把参数结构定义出来，方便后续做关卡包。

每关至少包含：
- `colorSet`（2/3/4/4）
- `headColorSwitchIntervalMs`（3000/2500/2000/1500/1000）
- `blockCountOnBoard`（随棋盘大小调整；16x16 建议 20~30 起步）
- `timeLimitSec`（可选：若复用限时模式）
- `scoreTarget`（可选：若复用标准/限时目标）
- `wrongColorRule`（strict/penalty）
- `specialItems`（可选：固定色道具、颜色反转等）

参考关卡意图（来自参考文档，结合本项目现状做字段化）：
1. 第1关：2 色 + 3 秒变色 + 低密度，学会“看蛇头颜色去吃发光块”
2. 第2关：3 色 + 2.5 秒变色 + 中密度，可加入“临时固定色道具（锁色 5 秒）”降低难度
3. 第3关：3 色 + 2 秒变色 + 高密度，可选增加“异色惩罚更重（僵直）”
4. 第4关：4 色 + 1.5 秒变色 + 方块可缓慢移动（后续）
5. 第5关：4 色 + 1 秒变色 + 无规律切换（后续）

## 需要的 UI / 反馈

- [ ] 蛇头颜色清晰可见（填充色 + 描边色 + 小光圈），避免被棋盘色块淹没
- [ ] HUD 展示当前颜色（以及下一次变色倒计时，建议用环形进度）
- [ ] 当前可吃颜色的方块发光：
  - [ ] 低成本实现：多层描边/多次绘制 + alpha 呼吸
  - [ ] 可选增强：粒子/后处理 Glow Pipeline（若成本可控）
- [ ] 颜色切换瞬间要有“咔哒一下”的反馈（音效/短闪）

## 开发任务拆分（建议按阶段交付可试玩里程碑）

### 阶段 0：规则与数据结构（不改 UI 也能跑）
- [ ] 定义 `HeadColorState`（当前颜色、下一次切换时间、切换策略）
- [ ] 在游戏循环中引入“蛇头颜色”概念，并保证重开/切模式/暂停恢复时状态一致
- [ ] 方块数据结构增加 `blockColor`，并能按颜色筛选“可吃集合”
- [ ] 引入 `wrongColorRule` 配置项（strict/penalty_*），默认 strict

验收：
- [ ] 开启该玩法分支后，只有同色方块会被判定为可吃
- [ ] 异色触碰按配置正确结算（默认 strict 失败）

### 阶段 1：发光提示与蛇头变色反馈（核心体验）
- [ ] 蛇头变色动画（短扩散光圈/描边闪）
- [ ] 同色方块发光（呼吸）
- [ ] HUD 展示当前颜色 + 变色倒计时

验收：
- [ ] 玩家无需读文字也能 3 秒内理解“现在该吃什么颜色”

### 阶段 2：关卡参数化（先做 1-2 关可玩）
- [ ] 定义 `ColorMatchLevelConfig`（至少包含：colorSet、切换间隔、方块数量、wrongColorRule）
- [ ] 做最小关卡选择（先内置 2 关即可）
- [ ] 关卡目标：优先复用现有“标准/限时”目标体系，避免大改结算

验收：
- [ ] 至少 2 关从易到难可连续游玩，且参数可快速调

### 阶段 3：扩展内容（可选）
- [ ] 临时固定色道具：吃到后锁色 N 秒
- [ ] 颜色反转道具：短时间同色/异色效果颠倒（仅在惩罚版模式下启用更合理）
- [ ] 异色陷阱方块：更重惩罚（仍建议配合惩罚版）

## 风险点与约束

- “严格同色可吃”若地图同色块过少，可能导致无解或强挫败：
  - [ ] 需要在生成逻辑上保证场上永远存在至少 K 个同色可吃块（例如 K=2~4）
- 发光效果要克制：避免像“强引导提示”压过整体视觉；建议只做轻微外发光和呼吸即可
- 该玩法分支应尽量做成独立开关/模式，不影响现有 5 个模式的稳定性
